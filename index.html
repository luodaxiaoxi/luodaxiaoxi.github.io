发表于 2019-08-22 分类于 随笔

笔记：一些需要注意的东西

1．压缩软件建议用winrar，压缩后的文件类型为.rar，.zip格式的的提交后系统自动解析可能会失效（改后缀无效）．

2．关于论文标题，不建议都使用问题题目，可以在标题中加入自己的方法，

例如 ：基于．．．的．．．模型．

3．摘要第一段是对下面各段的总述，不要太啰嗦（指说与题目无关的东西或者说一大段问题的背景等），也不要太短．

4．摘要的后面几段也建议尽量不要用千篇一律的　针对问题一，可以考虑将问题浓缩，写成：针对．．．的问题．

5．摘要建议采用总－分－总的的格式，先总述后分开对每一个问题做总结，最后把文章的亮点说出来（很关键）．

6．“关键词”三个字顶格写，不用空两格．关键词与正文空一行即可，不必一定要放在最后一行．

7．论文中的图片不能显示作者个人信息，也不能显示为屏幕截图，可先将保存的图片先放在一个word文档中，再从word文档中复制到论文中．

8．模型的假设处的符号使用英文全角的句号：“．”　．

9．符号说明处三线表可加中线，注意符号居中意义左对齐更美观．

10．公式结尾注意加上标点．

11．数据题的检验必不可少，机理题检验可体现在模型的求解中，注意可在摘要中提一下让评委知道文章做了这项工作．

12．模型的评价中优点至少３～４条，缺点不要回避（要有但是不要是本文的硬伤）．

GitHub Desktop上传文件至GitHub

首先在GitHub新建一个repository

​ 在GitHub建一个仓库，命名好加或者不加describe都行。

打开GitHub Desktop

​ 左上角Current repository，点开然后点击add会出现Clone repository，create new repository等，点击Clone repository并选择你刚刚创建的，然后等它克隆就行。

进入GitHub desktop文件夹

​ 会发现这里出现了你刚刚克隆的那个仓库，只需要把像上传的文件或文件夹添加进这个仓库，再打开GitHub desktop,就会发现这里面添加进来了。现在只需填写summary然后commit to master，再fetch origin就把你的东西上传到你的GitHub里了。

论文写作笔记

发表于 2019-08-05 分类于 随笔

论文写作

一、评阅原则

假设的合理性，建模的创造性；

结果的正确性，表述的清晰性。

  • 创造性：特别欣赏独树一帜，标新立异，但要合理。
  • 正确性：答案要有一定的精度，不能偏离“参考答案”很远。
  • 清晰性：摘要应理解为详细摘要，提纲挈领。

二、需要重视的问题

摘要

  • 字数控制在一页以内，半页以上；
  • 语言精炼，用词准确；
  • 阐述细致具体方法；
  • 列出主要结论；
  • 写出5-6个关键词。(方法、概念)

注：关键词用分号隔开，最后一个不加符号

问题的引出

  1. 对竞赛题中可能的模糊概念、条件作出必要的澄清
  2. 明确实际背景、已知条件、数据

问题的分析

​ 讨论什么问题，需要建立什么样的模型、用什么方法来求解做出一个定性的描述。

模型的假设

  • 严格、确切的数学语言表达
  • 假设应验证其合理性
  • 假设以3-6条为宜

例：假设题中所给数据准确无误(个别除外)

符号说明

三线表，列出全文通用符号

注：这里仅给出全文通用符号，其他符号在文中第一次出现时予以说明。

模型的建立与求解

模型的评价与检验

  1. 对结果的可靠性或误差分析
  2. 灵敏度和稳定度分析
  3. 对模型的客观评价，指出优越性和局限性
  4. 提出可能的深化，推广建议

三、多个模型的取舍问题

多个模型可选时，一至二个为宜；

实际做，可有多个，在写时选最好的，其他模型放在推广中。

SPSS回归分析8.3笔记

这里是采用了逐步回归的方式进行多元线性回归。

在分析->线性->统计 **中可以勾选置信区间**,看每个系数的区间估计，若R方很小且置信区间包含零点，则该模型要考虑可不可以用了。

关于置信区间。置信区间就是参数估计的一种，参数估计包括点估计和区间估计，一般来说点估计用的不多因为误差较大，而区间估计得到的就是一个置信区间，它一般会有一个置信度的问题，一般的软件(如MATLAB)都是默认为95%，如果某个参数的估计时，置信区间中存在零点，说明这个系数的估计是不可靠的。(见姜启源322页及323页改进的模型)

这个表主要看R的值。

​

R表示拟合优度（goodness of fit），它是用来衡量估计的模型对观测值的拟合程度。它的值越接近1说明模型越好。逐步回归发现R值逐渐变大，模型越来越好。在数学建模中一般要使R>0.8或0.9。

表Anova表示方差分析结果

​ 显著性即fig，当fig<0.05表示显著相关，当它<0.01表示高度相关。

​ F的值是回归方程的显著性检验，表示的是模型中被解释变量与所有解释变量之间的线性关系在总体上是否显著做出推断。若F>Fa(k,n-k-1)，则拒绝原假设，即认为列入模型的各个解释变量联合起来对被解释变量有显著影响，反之，则无显著影响。

​ 其中，k是自变量个数，n为样本容量，n-k-1为自由度。一般数理统计课本中都有F分布表，a表示的显著性水平（一般取0.05），在EXCEL中也能查表。在公式区输入：=FINV(0.05,4，6)，在单元格中即出现4.533677，表中的F值显著大于这个值，则认为各个解释变量对因变量有显著影响。
